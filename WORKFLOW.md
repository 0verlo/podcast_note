# 讲座笔记提取模板

## 使用方式

将本文件与讲座内容（文字稿、字幕文件、录音转写等）一同输入，要求 AI 按照以下流程和格式生成笔记。

---

## 分析流程

整体分三层递进：**粗切**（模块划分）→ **细切**（Q&A 映射）→ **提炼**（逐条写作）。每层有明确焦点，不混合处理。

### 0：评估源文件

用 `wc` 统计源文件字数，同时判断：

- **内容形式**：单人讲座 / 多人对谈 / 访谈
- **源文件类型**：书面文稿 / 口语转写

### 1：粗切 — 模块划分

通读全文，**只做结构判断，不做内容提炼**。目标是把原文切成可独立处理的模块：

- 识别话题转折点，将内容划分为若干模块；
- 判断哪些部分是核心论述，哪些是闲聊、过渡、广告等噪音（标记为可忽略）；
- 识别各参与者的身份和视角；
- 每个模块记录：**起止行号**、话题概述。

模块数量根据实际内容决定，不强行统一。

**2 万字以上的源文件**：粗切本身也需分段进行——先读前半部分划分模块，再读后半部分，衔接处检查是否有跨段话题。

### 2：细切 — 逐模块提取问题与行映射

**逐模块**进行，每次只聚焦一个模块对应的行范围。在模块内：

1. 找出以下类型的"问题"：
   - 讲者或对话者明确提出的问题；
   - 隐含在论述中的核心追问（即：这段话在回答什么问题）；
   - 听众可能自然产生的疑问。

2. 每个 Q&A 的 Q 标注其**对应的原文行范围**；

3. 估算**压缩比**：答案约为对应原文字数的 X%，记入 Q 列表。压缩比随内容密度浮动——信息密集的段落压缩少，重复或松散的段落压缩多。

问题密度参考：信息密度中等偏高的对谈/讲座，大约 **800—1000 字产生 1 个议题**。可据此估算合理的问题总数，避免过多或过少。

### 3：Q 列表输出与覆盖审计

细切完成后，输出第一轮结果到笔记.md：

```
## 模块一：[模块标题]

**Q:** [问题]（行 X1–Y1，约为原文60%字数）

**Q:** [问题]（行 X2–Y2，约为原文50%字数）

## 模块二：[模块标题]

**Q:** [问题]（行 X3–Y3，约为原文45%字数）
```

逐 Q 对应回原文，扫描未被任何 Q 覆盖的段落：

- 属于核心论述的，补充对应 Q；
- 属于闲聊、过渡、重复的，确认可忽略；
- 属于跨模块衔接处的遗漏，补齐。

确认 Q 列表完整后，再进入提炼。

### 4：提炼 — 逐 Q&A 写作

根据笔记.md已有的每条 Q 逐条处理，做下列操作：

1. 取 Q 列表标注的行范围，上下各扩展 2 行，重新读一遍对应行数的原文；
2. 复评压缩比——如果阅读后判断理论密度过高，需要调整，更新 Q 列表中的压缩比；
3. 按压缩比写答案。

这一层是写作质量的核心。

**基本原则：**

- 组织语言提炼总结原文内容，确保表述与原文逻辑一致；
- 保留具体例子、数据、人名、书名、术语原文、公式符号等关键细节，这些是答案的"骨肉"；
- 提炼的压缩率不是平均发生的，对承担论证、转折、定义、因果等功能的内容，默认低压缩率，优先保留论证骨架。
- 如果讲者显式标记了某段为论证依据，需要注意保留这部分的描述；
- 如果讲者对某个问题只给出了部分答案或明确表示不确定，如实标注，不补充推断；
- 如果同一问题有多个不同视角或争议，分别列出。

**多人对谈时额外关注：**

- **发言归属**：标注谁说了什么（"Krieger 认为…"、"Dan 补充…"），不要把不同人的观点揉成无主语的总结；
- **对话动态**：保留"立论→补充→反例→再回应"的互动结构，这本身就是内容的一部分。

**口语转写时额外关注：**

- filler words（you know, like, I mean）不携带语义，切分句意时需跳过；
- 外文术语或书名可能会被错误转写——根据上下文还原原文，无法确认的保留转写并加注存疑；
- 口语中的插入语容易被误读为修饰语——遇到结构不确定的句子，先还原为"讲者想表达的完整意思是什么"，再组织措辞。

### 5：准确性审计

逐条 Q&A 回查其对应的原文行范围，校验：

- 事实是否准确（人名、团队、数据、因果关系）；
- 是否添加了原文没有的内容（幻觉）；
- 发言归属是否正确（多人对谈时，观点是否归对了人）；
- 语义是否被扭曲（尤其注意口语转写中的歧义句）；
- 外文术语或书名是否转写错误或被遗漏（转写的问题是否保留到了笔记）；
- 中文表述是否流畅、符合逻辑，有无生硬或歧义的措辞。

发现错误时，**将审计范围扩大到同模块的其他 Q&A**——同批次生成的内容共享注意力水平，一处出错通常意味着相邻内容也有风险。

### 6：标注概念链接

每个模块的最后一个 Q&A 之后、分隔线之前，添加一行概念链接，指向 `_concepts/` 下的分类 hub 页。

**概念分类与粒度**

分类文件由用户预设和维护，workflow 只负责将讲座中出现的概念归入对应分类，不负责分类体系的设计和调整。

概念根据讲座内容总结，按大学课程大纲的粒度选取。当前分类文件为四个：

| 分类文件 | 收录内容 | 示例 |
|---|---|---|
| `人物.md` | 思想家、学者 | 阿尔都塞、福柯、德勒兹 |
| `学派与方法.md` | 思想流派、研究方法论、思想传统 | 批判理论、结构主义、谱系学、儒学 |
| `核心概念.md` | 学科通用的关键术语 | 意识形态、规训、主体性、他者化 |
| `精神分析.md` | 精神分析专属术语与概念 | 能指、享乐、对象a、升华、圣状 |

表内示例不构成限制，具体概念条目随讲座内容按需增加，归入对应分类即可。

选取原则：
- 只选课程大纲级别的关键词，不选某篇讲座的专属说法
- 子概念归入上级概念（如"质询""意识形态国家机器"归入"意识形态"）
- 一个概念可在多个模块出现

**链接格式**

按 人物 → 学派与方法 → 核心概念 的顺序排列：

```
概念：[[人物#福柯]], [[学派与方法#批判理论]], [[核心概念#规训]]
```

**Hub 页维护**

`_concepts/` 下每个分类文件以 `## 概念名` 为标题，下挂指向具体模块的跳转链接：

```markdown
## 福柯

- [[bili/01_批判理论的文化边界：从欧陆到东亚社会#模块一：欧洲批判理论的原初语境与预设读者|欧洲批判理论的原初语境与预设读者]]
```

新增笔记时，往对应概念的 `##` 下追加条目；如果出现新概念，在对应分类文件中新建 `##` 标题。

### 7：整理待深挖线索

通读全部 Q&A 后，找出以下类型的内容，归入 "L: What's Left" 区块：

- 讲者提到但未展开的书目、论文、人物、概念；
- 讲者自己承认没有完全理解或尚未探究的问题；
- Q&A 中出现的逻辑跳跃或未说清楚的环节；
- 你认为有必要进一步查证或深入阅读的方向。

线索应按类别分组，推荐分类：

- **书目**：讲者提到的具体著作、文章
- **脉络**：值得进一步了解的人物及其关系
- **概念**：被提及但未充分展开的理论或术语
- **话题**：讲者跳过、承认不懂、或明确留待以后的内容

---

## 最终输出格式

```
# 讲座笔记：[标题或主题]
> 来源信息（主讲人、节目名、日期等，按实际填写）

---

## 模块一：[模块标题]

**Q:** [问题]（行 X1–Y1，约为原文60%字数）
**A:** [答案]

**Q:** [问题]（行 X2–Y2，约为原文80%字数）
**A:** [答案]

概念：[[人物#xxx]], [[学派与方法#xxx]], [[核心概念#xxx]]

---

## 模块二：[模块标题]

**Q:** [问题]（行 X3–Y3，约为原文50%字数）
**A:** [答案]

概念：[[核心概念#xxx]]

---

## L: What's Left

- [书目] 作者，《书名》— 说明
- [书目] 作者，《书名》— 说明

- [脉络] **人物** — 说明

- [概念] **术语** — 说明

- [话题] 描述 — 说明

```

## 注意事项

- 模块划分以话题为准，不以时间或段落数量为准；
- Q&A 的数量服从内容本身，不追求数量均衡；
- 闲聊、主持过渡语、音频噪音等内容直接忽略，不计入笔记；
- 如果讲座内容本身结构混乱，可在笔记开头加一行说明，标注哪些部分被舍弃及原因；
- What's Left 区块是笔记的重要组成部分，不应省略。它的价值在于：讲座听完即忘，但线索可以指向下一步真正深入的阅读。
