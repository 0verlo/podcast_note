# 讲座笔记提取模板

## 使用方式

将本文件与讲座内容（文字稿、字幕文件、录音转写等）一同输入，要求 AI 按照以下流程和格式生成笔记。

---

## 分析流程

### 第零步：评估源文件规模

先用 `wc` 统计源文件字数。根据字数决定处理策略：

- **2 万字以内**：可一次通读处理；
- **2 万字以上**：按话题切换点分段，逐段分析后合并，避免信息遗漏。

### 第一步：识别内容结构

通读全文，判断以下信息：

- 讲座是否有明显的主题切换（话题转折点）；
- 哪些部分是核心论述，哪些是闲聊、过渡或无关噪音；
- 讲者的身份和视角（决定哪些内容更权威、更值得记录）。

将内容划分为若干模块，每个模块对应一个相对独立的话题。模块数量根据实际内容决定，不必强行统一。

### 第二步：提取问题

在每个模块内，找出以下类型的"问题"：

- 讲者或对话者明确提出的问题；
- 隐含在论述中的核心追问（即：这段话在回答什么问题）；
- 听众可能自然产生的疑问。

问题密度参考：此类信息密度中等偏高的对谈/讲座，大约 **800—1000 字产生 1 个议题**。可据此估算合理的问题总数，避免过多或过少。

### 第三步：提炼答案

针对每个问题，从讲座内容中提炼答案。原则如下：

- 用自己的语言重新组织，不直接照搬原文；
- 保留具体例子、数据、人名、书名等关键细节，这些是答案的"骨肉"；
- 如果讲者对某个问题只给出了部分答案或明确表示不确定，如实标注，不补充推断；
- 如果同一问题有多个不同视角或争议，分别列出。

### 第四步：标注概念链接

每个模块的最后一个 Q&A 之后、分隔线之前，添加一行概念链接，指向 `_concepts/` 下的分类 hub 页。

**概念分类与粒度**

概念根据讲座内容总结，按大学课程大纲的粒度选取。分类文件固定为三个：

| 分类文件 | 收录内容 | 示例 |
|---|---|---|
| `人物.md` | 思想家、学者 | 阿尔都塞、福柯、德勒兹 |
| `学派与方法.md` | 思想流派、研究方法论、思想传统 | 批判理论、结构主义、谱系学、儒学 |
| `核心概念.md` | 学科通用的关键术语 | 意识形态、规训、主体性、他者化 |

表内示例不构成限制，具体概念条目随讲座内容按需增加，归入对应分类即可。

选取原则：
- 只选课程大纲级别的关键词，不选某篇讲座的专属说法
- 子概念归入上级概念（如"质询""意识形态国家机器"归入"意识形态"）
- 一个概念可在多个模块出现

**链接格式**

按 人物 → 学派与方法 → 核心概念 的顺序排列：

```
概念：[[人物#福柯]], [[学派与方法#批判理论]], [[核心概念#规训]]
```

**Hub 页维护**

`_concepts/` 下每个分类文件以 `## 概念名` 为标题，下挂指向具体模块的跳转链接：

```markdown
## 福柯

- [[bili/01_批判理论的文化边界：从欧陆到东亚社会#模块一：欧洲批判理论的原初语境与预设读者|欧洲批判理论的原初语境与预设读者]]
```

新增笔记时，往对应概念的 `##` 下追加条目；如果出现新概念，在对应分类文件中新建 `##` 标题。

### 第五步：整理待深挖线索

通读全部 Q&A 后，找出以下类型的内容，归入 "L: What's Left" 区块：

- 讲者提到但未展开的书目、论文、人物、概念；
- 讲者自己承认没有完全理解或尚未探究的问题；
- Q&A 中出现的逻辑跳跃或未说清楚的环节；
- 你认为有必要进一步查证或深入阅读的方向。

线索应按类别分组，推荐分类：

- **书目**：讲者提到的具体著作、文章
- **脉络**：值得进一步了解的人物及其关系
- **概念**：被提及但未充分展开的理论或术语
- **话题**：讲者跳过、承认不懂、或明确留待以后的内容

---

## 输出格式

```
# 讲座笔记：[标题或主题]
> 来源信息（主讲人、节目名、日期等，按实际填写）

---

## 模块一：[模块标题]

Q: [问题]
A: [答案]

Q: [问题]
A: [答案]

概念：[[人物#xxx]], [[学派与方法#xxx]], [[核心概念#xxx]]

---

## 模块二：[模块标题]

Q: [问题]
A: [答案]

概念：[[核心概念#xxx]]

---

## L: What's Left

- [书目] 作者，《书名》— 说明
- [书目] 作者，《书名》— 说明

- [脉络] **人物** — 说明

- [概念] **术语** — 说明

- [话题] 描述 — 说明

```

## 注意事项

- 模块划分以话题为准，不以时间或段落数量为准；
- Q&A 的数量服从内容本身，不追求数量均衡；
- 闲聊、主持过渡语、音频噪音等内容直接忽略，不计入笔记；
- 如果讲座内容本身结构混乱，可在笔记开头加一行说明，标注哪些部分被舍弃及原因；
- What's Left 区块是笔记的重要组成部分，不应省略。它的价值在于：讲座听完即忘，但线索可以指向下一步真正深入的阅读。

## Commit 格式

```
<一句话概述>

[Notes]     ← 笔记增删改
[Raw]       ← 原始素材增删
[Chore]     ← 工程及结构相关
[Format]    ← 内容格式/样式调整
```

按需使用相关分类，无变更的分类省略不写。

范例：

```
初始化笔记仓库

[Notes]
1.add 13_语言学相关对谈
2.add 37_相似性与关联

[Raw]
1.add 13_语言学相关对谈
2.add 37_相似性与关联

[Chore]
1.add WORKFLOW.md 笔记提取模板
2.add README.md
```
